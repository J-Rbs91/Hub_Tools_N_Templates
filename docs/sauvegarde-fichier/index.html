Sauvegarde dans un fichier — en toute transparence

Il existe deux façons d'enregistrer et de retrouver le paramétrage de votre magasin : par Google Drive (partagé entre tous les postes) ou par fichier (la présente page). Aucune n'est meilleure : c'est vous qui choisissez. Cette page décrit, sans rien cacher, la façon par fichier — sans aucun compte Google : comment l'enregistrer (clé USB ou ordinateur), ce que le fichier contient exactement, et où vivent vos données.

Aucun compte, aucune installation · quelques secondes  ·  Voir plutôt la façon « Google Drive »

Où sont vos données, en réalité ?

  • Tout reste dans le navigateur de cet ordinateur. Vos coordonnées, les paramètres de caisse et l'historique de clôture sont mémorisés localement (dans le stockage du navigateur), sur ce poste uniquement. Rien n'est envoyé à un serveur.
  • Le fichier n'est créé que si vous cliquez. Tant que vous ne cliquez pas sur « Sauvegarder dans un fichier », aucun fichier n'existe et rien ne sort du navigateur.
  • Vous décidez où il va. Le fichier est téléchargé comme n'importe quel document : c'est vous qui le rangez (clé USB, dossier, e-mail à vous-même…).
  • C'est un fichier texte lisible. Vous pouvez l'ouvrir avec le Bloc-notes pour voir, de vos yeux, exactement ce qu'il contient — aucun format secret.

Comment ça fonctionne — un paramétrage par poste

Avec la façon « fichier », le paramétrage vit sur l'ordinateur où vous le faites. Chaque poste a donc sa propre copie. Il est important de bien comprendre ce que cela implique — voici les choses telles qu'elles sont :

Les informations qui ne changent jamais : rien à entretenir

Le nom, l'adresse, le téléphone et l'e-mail du magasin sont des informations statiques : elles ne bougent quasiment jamais. Une fois posées sur un poste (par la saisie, ou en restaurant le fichier), il n'y a plus rien à faire ni à mettre à jour. C'est le cas d'usage le plus courant, et il fonctionne très bien ainsi.

La contrepartie : ce qui n'est pas partagé entre les postes

  • Comme le fichier reste sur la machine, ce qui devrait être commun à tous les postes n'est pas partagé automatiquement. Si vous modifiez le profil du magasin sur un poste, les autres postes ne le savent pas.
  • Pour aligner les postes, il faut refaire une sauvegarde sur le poste modifié, puis une restauration sur les autres. Une manipulation manuelle, à votre main.
  • Les paramètres de caisse et l'historique de clôture sont, de toute façon, toujours propres à chaque poste (quelle que soit la façon choisie) : le fichier sert surtout à les transférer lors d'un changement de poste, pas à les synchroniser en continu.

Si vous préférez au contraire que le profil du magasin se mette à jour tout seul sur tous les postes, c'est la façon Google Drive qu'il faut choisir. À vous de voir ce qui convient le mieux à votre organisation.

Ce que contient le fichier — exactement

Le fichier porte un nom du type sauvegarde-magasin-votre-magasin-2026-06-10.json. Voici précisément ce qu'on y trouve : 

{
  "app": "hub-magasin",
  "version": 1,
  "exportedAt": "2026-06-10T18:32:00.000Z",
  "profil": {
    "nom": "Optique du Centre",
    "adresse": "12 rue des Lunettes, 34000 Ville",
    "tel": "04.00.00.00.00",
    "mail": "contact@votremagasin.fr"
  },
  "reserve": 53.80,
  "fondCible": 200.00,
  "caisse": [
    { "date": "2026-06-09", "total": 412.50, "ecart": 0 }
  ]
}

Exemple : vos vraies valeurs à la place de celles-ci. Le bloc caisse contient au plus les 15 derniers jours.

Dans le fichierCe que c'est
profil Les coordonnées du magasin saisies dans « Mon magasin » : nom, adresse, téléphone, e-mail.
reserve, fondCible La réserve de monnaie et le fond de caisse cible utilisés par la Clôture de Caisse.
caisse L'historique de clôture des 15 derniers jours : par jour, des montants (total espèces, écart…). Ce sont des chiffres de caisse, pas des informations sur des personnes.
app, version, exportedAt Des repères techniques : le type de fichier, sa version et la date d'export, pour que la restauration reconnaisse un fichier valide.

Ce que le fichier ne contient jamais

  • ✘ Aucune donnée patient — rien des demandes ou ordonnances (noms de clients, prescriptions, motifs…). Ces saisies ne sont d'ailleurs jamais conservées : elles servent à générer le document, puis disparaissent.
  • ✘ Pas l'adresse de synchronisation Drive (le cas échéant), ni aucun mot de passe.

Sauvegarder (créer le fichier)

  1. Ouvrir « Mon magasin »
    Sur le hub, cliquez sur ⚙ Mon magasin, puis descendez jusqu'à la section « Façon 2 — par fichier ».
  2. Cliquer sur « Sauvegarder dans un fichier »
    Un fichier .json se télécharge aussitôt (dans vos « Téléchargements », comme un document classique).

    Ce qui se passe : le navigateur rassemble le profil, les paramètres de caisse et l'historique (15 jours) déjà présents sur ce poste, et les écrit dans ce fichier. Aucune connexion, aucun envoi — le fichier est fabriqué sur votre ordinateur.

  3. Mettre le fichier en lieu sûr
    Copiez-le sur une clé USB, dans un dossier sauvegardé, ou envoyez-le-vous par e-mail. Refaites une sauvegarde de temps en temps (le fichier le plus récent contient l'historique le plus à jour).

Restaurer (réinjecter le fichier)

Utile en cas de changement de poste, de vidage du navigateur, ou pour recopier le paramétrage sur un autre ordinateur.

  1. Ouvrir « Mon magasin » sur le poste à restaurer
    Section « Façon 2 — par fichier », cliquez sur « Restaurer un fichier… ».
  2. Choisir votre fichier .json
    Sélectionnez le fichier de sauvegarde. L'outil vérifie d'abord qu'il s'agit bien d'une sauvegarde du magasin avant d'agir.
  3. Vérifier le résultat
    Un message confirme la restauration. Concrètement :
    • le profil et les paramètres de caisse sont remplacés par ceux du fichier ;
    • l'historique de clôture est fusionné (les jours du fichier s'ajoutent à ceux déjà présents, en gardant les 15 jours les plus récents).

    Bon à savoir : tout se passe sur ce poste, dans le navigateur. Le fichier n'est pas « ouvert » par un service extérieur — c'est l'outil, dans votre navigateur, qui le lit.

Votre responsabilité, une fois le fichier créé

  • À partir du moment où vous déposez le fichier quelque part (clé USB, e-mail, cloud personnel), il quitte la protection de l'outil et devient un document comme un autre : rangez-le en lieu sûr, en interne.
  • Il ne contient aucune donnée patient. Il contient seulement les informations de votre magasin que vous avez souhaité renseigner et les montants de caisse que vous enregistrez : traitez-le avec le même soin que vos documents internes.
  • Pour repartir de zéro sur un poste, il suffit de vider les données du site dans le navigateur ; rien ne subsiste ailleurs.
Un doute ? Ouvrez le fichier avec le Bloc-notes : vous y verrez, en clair, exactement les informations décrites ci-dessus. C'est tout ce qu'il contient.
