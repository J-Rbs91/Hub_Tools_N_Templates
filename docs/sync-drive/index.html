Retrouver votre paramétrage sur tous vos postes

Il existe deux façons d'enregistrer et de retrouver le paramétrage de votre magasin (nom, adresse, téléphone, e-mail) : par Google Drive (la présente page) ou par fichier sur clé USB / ordinateur. Aucune n'est meilleure : c'est vous qui choisissez. Cette page décrit la façon Google Drive : le paramétrage est mémorisé dans votre propre Drive, ce qui permet de le retrouver automatiquement sur tous les postes connectés au compte Google du magasin — sans avoir à le refaire poste par poste.

Durée : environ 5 minutes, une seule fois  ·  Voir plutôt la façon « par fichier »

Avant de commencer :

Ce type de manipulation peut sembler technique, voici donc exactement ce qui se passe :

  • Vos données restent chez vous. Elles sont écrites dans un dossier de votre Google Drive (créé automatiquement, nommé Hub_Facilities). Aucun serveur tiers, aucun éditeur extérieur n'y a accès.
  • Vous restez propriétaire et maître. Le petit programme est créé sous votre compte Google ; vous pouvez le supprimer ou couper l'accès à tout moment.
  • Aucune donnée patient n'est concernée. Seules les coordonnées du magasin sont synchronisées — jamais les informations des clients.
  • Le code est court et lisible (vous le collez vous-même à l'étape 2) : il ne fait que lire et écrire un fichier dans ce dossier.

Les étapes

  1. Ouvrir l'outil de script de Google
    Sur l'ordinateur, connectez-vous au compte Google du magasin. Ouvrez un nouvel onglet, allez sur script.google.com puis cliquez sur Nouveau projet.

    Pourquoi ? « Apps Script » est l'outil officiel de Google qui permet à votre compte d'automatiser de petites tâches sur votre propre Drive.

  2. Coller le programme fourni
    Sélectionnez tout le texte déjà présent, supprimez-le, puis collez ce code (bouton Copier). Enregistrez avec l'icône disquette. 
    var DOSSIER = 'Hub_Facilities';
var FICHIER = 'profil-magasin.json';

function doGet(e) {
  var p  = e.parameter || {};
  var cb = p.callback;
  var res;
  try {
    if (p.action === 'save') {
      JSON.parse(p.data || '{}');   // vérifie que la donnée est valide
      ecrire_(p.data || '{}');
      res = { ok: true };
    } else {
      res = { ok: true, data: lire_() };
    }
  } catch (err) {
    res = { ok: false, error: String(err) };
  }
  var json = JSON.stringify(res);
  if (cb) {
    return ContentService.createTextOutput(cb + '(' + json + ')')
      .setMimeType(ContentService.MimeType.JAVASCRIPT);
  }
  return ContentService.createTextOutput(json)
    .setMimeType(ContentService.MimeType.JSON);
}

function dossier_() {
  var it = DriveApp.getFoldersByName(DOSSIER);
  return it.hasNext() ? it.next() : DriveApp.createFolder(DOSSIER);
}
function fichier_() {
  var it = dossier_().getFilesByName(FICHIER);
  return it.hasNext() ? it.next() : null;
}
function lire_() {
  var f = fichier_();
  return f ? f.getBlob().getDataAsString() : '';
}
function ecrire_(contenu) {
  var f = fichier_();
  if (f) { f.setContent(contenu); }
  else { dossier_().createFile(FICHIER, contenu, 'application/json'); }
}
    Que fait ce programme, en clair ?
    Il sait faire deux choses, et rien d'autre : « enregistrer » (écrire vos coordonnées dans le fichier profil-magasin.json) et « lire » (renvoyer ce qui y est enregistré). Le dossier et le fichier sont créés tout seuls la première fois. Il n'accède à aucun autre fichier de votre Drive.
  3. Publier en « application web »
    En haut à droite : Déployer → Nouveau déploiement. Cliquez sur la roue dentée et choisissez Application web. Réglez ensuite :
    • Exécuter en tant que : Moi
    • Qui a accès : Tout le monde

    Pourquoi ces réglages ? « Exécuter en tant que moi » fait que le programme agit dans votre Drive (et nulle part ailleurs). « Tout le monde » signifie seulement que l'outil peut joindre cette adresse : l'adresse est longue et connue de vous seul, elle joue le rôle de clé — ne la partagez qu'en interne.

    Le choix « Tout le monde » n'apparaît pas ?

    Certains comptes Google gérés par une entreprise (comptes professionnels « Workspace ») ne proposent que « Moi uniquement » ou « Tous les utilisateurs du domaine », jamais « Tout le monde ». La synchronisation ne peut alors pas fonctionner avec ce compte.

    L'astuce : créez gratuitement une adresse Gmail personnelle dédiée au magasin (par exemple monmagasin.sync@gmail.com) sur gmail.com, puis refaites les étapes 1 à 5 en étant connecté à cette adresse. Sur un compte Gmail personnel, le choix « Tout le monde » redevient disponible. N'utilisez ce compte que pour cette synchronisation.

  4. Autoriser votre propre programme
    Google vous demande d'autoriser l'accès à votre Drive : acceptez avec le compte du magasin.

    Un écran d'avertissement « Application non vérifiée » va apparaître — c'est normal et sans danger ici. Google l'affiche pour tout script personnel qu'il n'a pas audité ; comme c'est vous qui venez de le créer, vous pouvez continuer en toute confiance.

    1) L'écran d'avertissement :

    Google n'a pas validé cette application
    L'application demande l'accès à des informations sensibles de votre compte Google. Tant que le développeur (contact@votremagasin.fr) n'a pas fait valider cette application, vous ne devriez pas l'utiliser.
    Masquer les paramètres avancés Revenir à la sécurité
    Continuez uniquement si vous comprenez les risques et faites confiance au développeur (contact@votremagasin.fr).
    Accéder à Projet sans titre (non sécurisé)

    ✔ Cliquez sur « Paramètres avancés » (en bas à gauche) si le bas est masqué, puis sur le lien bleu « Accéder à Projet sans titre (non sécurisé) ».
    ✘ N'appuyez pas sur le bouton bleu « Revenir à la sécurité ».

    2) L'écran d'autorisation (fond sombre) :

    « Projet sans titre » souhaite accéder à votre compte Google
    contact@votremagasin.fr
    Cette application n'a pas été validée par Google. Comme elle demande l'accès à votre compte Google, ne continuez que si vous connaissez et faites confiance au développeur.
    Lorsque vous autoriserez cet accès, « Projet sans titre » pourra :
    Voir, modifier, créer et supprimer tous vos fichiers Google Drive.
    AnnulerContinuer

    ✔ Cliquez sur « Continuer » (« Continue » en anglais) pour terminer l'autorisation.

    Pourquoi Google annonce « tous vos fichiers Drive » ? C'est une limite technique d'Apps Script : il réclame l'autorisation Drive complète. En pratique, le script que vous avez collé ne lit et n'écrit qu'un seul fichier (profil-magasin.json, dans le dossier Hub_Facilities) — vous pouvez le vérifier dans le code de l'étape 2.

  5. Copier l'adresse obtenue
    À la fin, Google affiche la fenêtre « Nouveau déploiement » contenant l'URL d'application web (elle se termine par /exec) : c'est elle que l'outil utilisera.
    Nouveau déploiement
    Mise à jour du déploiement effectuée.
    ID de déploiement
    AKfycb…qpbsMhCopier
    Application Web — URL
    https://script.google.com/macros/s/AKfycb…/execCopier
    OK

    ✔ Cliquez sur « Copier » sous l'URL (celle qui finit par /exec), puis collez-la à l'étape suivante. (L'« ID de déploiement » ne sert pas.)

  6. Coller l'adresse dans l'outil
    Revenez sur le hub, ouvrez ⚙ Mon magasin, collez l'URL dans le champ URL de synchronisation, puis Enregistrer. Vos coordonnées partent alors dans votre Drive. Sur chaque autre poste, collez la même URL et cliquez sur Récupérer depuis le Drive : tout réapparaît.

Bon à savoir

  • Toute personne qui possède cette URL peut lire et modifier ces coordonnées : gardez-la en interne.
  • Pour tout arrêter : supprimez le projet sur script.google.com, ou videz le champ URL dans « Mon magasin ». Vos outils continuent de fonctionner en local.
Un doute ou un blocage à une étape ? Notez le numéro de l'étape et le message affiché : c'est suffisant pour être dépanné rapidement.
